GIT AND GITHUB TUTORIAL

What is Git and GitHub?

GitHub allows users to share and collaborate with others and can be used publicly or privately to create private projects.

Git can make the single group code workflow much simpler by combining branches. You can work anywhere by simply copying the source code from the repository or copying a revision from the repository, or a branch from the repository and easy easy in product deploy.

Step 1: Install Git and create a GitHub account

-Choose the right operating system for your desktop and install Git

-Pick a username, enter your email address and a password, and click Sign up for GitHub. Once you are in, it will look something like this:

Step 2: Create a new repository

-Enter name for your repository (username.github.io: it’s your hosting repository) and click Create Repository

Step 3: Create git configuration

-Open Terminal program to setup config for git on your computer.

-Type git to check Git’s version in your PC.

-Type git config to see config in your PC.

-Type git config -l to list all the files.

-Type git config –global user.name “your GitHub username” to setup your username.

-Type git config –global user.email “your GitHub email” to setup your email.

-It will look like:

Step 4: Connection between Git and GitHub

-Type git clone URL to download your repository URL to your computer:

-Type cd .. to change the your local in your computer:

-Type git add. to add a file to push to your repository.

-Type commit -m “write something that you commit” to record changes to your repository.

-Type git push to push a file to your repository.

-Type git pull to edit in the local (your PC) and/or in the online (your repo):

  • Connection with your friends

-Choose Setting

-Choose Manage access

-Click Invite a collaborator and type friend’s username.

Step 5: Create a branch and solve conflict

  • Create a branch

-Type git branch <branchname> (new branch)

-Type git checkout newbranch.

-Create a file conflict.txt (conflict file)

-Use git add . , git commit and git push origin newbranch.

  • Solve conflict

-Type git checkout main (main is default branch in your repo)

-Create a file same name conflict.txt

-Type git add . , git commit and type git push merge newbranch.

-At this point git will complain and suggest you to resolve conflict by accepting which version of the file conflict.txt and then resolve conflict use git add ., git commit and git push.
